No emojis in any deliverable — spreadsheets, audit reports, Notion pages, briefs, edit reviews, Slack-bound summaries, anything that goes into the user's workflow.

**Why:** User asked me to strip emojis from the locale audit sheet after I used checkmarks/warning symbols/section icons as visual markers. They prefer plain text in working documents. The default project guideline already says no emojis unless explicitly requested — the spreadsheet emoji use was a violation worth not repeating.

**How to apply:** When generating sheets, Notion pages, content briefs, edit reviews, audit reports, or any other deliverable, use plain text section headers (e.g. "GOOD", "PROBLEMS", "P0") rather than ✅/❌/⚠️/📊/etc. Plain typography only. This applies even to "visual" markers that feel scannable — they aren't worth the noise.
